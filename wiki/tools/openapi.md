# OpenAPI

**OpenAPI** is a machine-readable specification format for HTTP APIs. It is not usually branded as AI [[spec-driven-development|SDD]], but it is one of the clearest mature examples of spec-first development working in practice.

An OpenAPI document can describe endpoints, request and response shapes, authentication, errors, and examples. Tooling can then generate documentation, clients, server stubs, mocks, validation, tests, and security checks.

For the AI ecosystem map, OpenAPI matters because it shows what "human-readable and machine-usable" artifacts can do. Many SDD tools are trying to generalize this pattern beyond APIs: the spec becomes a reviewed source artifact that other tools can act on.

**Limits** - OpenAPI is focused on API contracts, not full product behavior, architecture decisions, or implementation plans.
