# GitHub Spec Kit

**GitHub Spec Kit** is an open-source toolkit for [[spec-driven-development|spec-driven development]] with AI coding agents. It turns a feature idea into a sequence of artifacts: project principles, a behavior spec, a technical plan, tasks, and implementation steps.

Its workflow usually separates "what and why" from "how." The specify phase captures the user journey and success criteria; the plan phase adds architecture, stack, and constraints; tasks break the work into reviewable chunks.

Spec Kit is important less as a single tool and more as a signal: AI coding workflows are moving from one-shot prompts toward versioned, reviewable artifacts that agents can execute. It also includes optional clarification, checklist, convergence, and issue-creation steps.

**Risk** - more artifacts can mean more review burden. Spec Kit is strongest when specs stay concise and grounded in repository evidence.
