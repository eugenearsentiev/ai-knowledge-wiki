# Model Classifications

AI models are best classified across several axes, not by a single label. The same model can be a [[llm|language model]], open-weight, multimodal, large, cloud-hosted, and optimized for coding.

**Foundation models** are broad base models adapted through prompting, [[rag|retrieval]], [[fine-tuning|fine-tuning]], or tool use. Examples include [[gpt|GPT]], [[claude|Claude]], [[gemini|Gemini]], [[llama|Llama]], [[bert|BERT]], and [[stable-diffusion|Stable Diffusion]].

**By family** — [[llm|LLMs]] generate text and code. VLMs combine images plus language. SLMs optimize for cost, latency, local use, or narrow tasks. Embedding models create vectors for [[embeddings|semantic search]] and [[rag|retrieval]]. Classifiers predict labels.

**By openness and deployment** — proprietary API, open-weight, or open-source; cloud, self-hosted, local, or on-device. Parameter count is only a rough proxy; product fit also depends on privacy, control, license, latency, and cost.

**By evaluation fit** — [[model-evals|Evals]] compare models against the actual job: benchmarks, preference tests, automated judge checks, safety audits, cost, and regressions. A smaller model can win if it clears the product-specific eval suite.
